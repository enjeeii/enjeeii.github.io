Python | 浅谈模块导入机制

发表于 2024-08-21 更新于 2024-09-21 分类于 编程语言

为了方便维护及复用等需求，Python 将各种定义存入一个个脚本文件中，供需要时导入使用
这些脚本文件就是一个个模块，模块（们）经过合理的组织成为一个个包
包和模块即是 Python 用来管理、组织源代码的手段

包和模块

包和模块在 Python 中可以类比于文件系统中的目录和文件，其共同组成了 Python 程序的代码架构

模块

模块是包含 Python 定义和语句的文件，其文件名为模块名加上 .py 后缀名，其中定义的名字可以被其他模块导入使用
在模块内部，通过全局变量 __name__ 可以获取模块的名字，特别地，在命令行直接执行脚本时 __name__ 属性会被赋值为字符串 __main__
模块中的可执行语句和各类定义语句会在第一次被导入时，或者直接运行时执行
每个模块拥有自己的命名空间，被用作模块内所有定义的全局命名空间；在模块内导入其他模块时，会将导入的模块通过对应的名字绑定到全局命名空间中

关于命名空间：名字到对象的映射集合
一个命名空间的作用域是这个命名空间中的名字所能生效的区域，常见有：

  1. 最内层的作用域，包含局部名称（通常是函数）
  2. 外层作用域，包括外层函数等 “非全局、非局部” 的名字
  3. 全局作用域，包含当前模块的全局名称
  4. 最外层的作用域，是内置名称的命名空间

基于以上，对于两个改变作用域的关键字：
global 重新绑定到全局名字
nonlocal 重新绑定到非局部的外层作用域中的名字（排除当前最内层的作用域逐步向外搜，搜到谁是谁）

包

包作为模块的集合（也可以拥有子包是包的集合），可以有效地对模块进行分类和管理
在 Python 包有两种形式：常规包和命名空间包；而无论哪种类型的包，都拥有一个 __path__ 的属性，其会影响模块的搜索路径
特别地，如果一个模块拥有 __path__ 属性，则会被视为一个包

常规包
一个带有 __init__.py 文件（可以是一个空文件）的目录
在导入常规包时，__init__.py 文件会被执行，其中的定义会被绑定到包的全局命名空间中，其中有两个特殊的属性需要注意：

  1. __path__
    这个属性是一个列表并且可以修改，并且可修改
    在 __init__.py 脚本执行前，这个属性被初始化为脚本所在目录
    上面说到这个属性会影响包内的模块搜索路径，即在搜索这个包中的模块或子包时，会在这个属性所列出的目录中搜索
  2. __all__
    这个属性是一个可定义的列表（或元组，建议使用元组）
    具有提示包导入时那些名字可以被导入的作用，这个提示具体体现在使用 from package import * 语法导入时，会导入这个列表里的名字
    另外，这个属性也可以在模块中出现，具有同样的作用

命名空间包

PEP 420 引入

不带 __init__.py 文件的目录，并且可以由多个同名的目录共同组成一个包
我对于命名空间包的理解，类似于 C++ 中的 namespace 可以在任何地方出现，不同位置出现的同名 namespace 也视为同一个名字空间

导入模块

模块导入功能用以支持导入包和模块中的代码和定义来实现复用
一般来说，有两种导入方式：

  1. import ...
  2. from ... import ...

Python 通过完整限定名称来导入模块，这是一个可以带点号的指向模块的路径名，路径上的每一个部分都会被尝试导入，并将记录到 sys.modules 中缓存
例如对于 foo.bar.baz 将会导入：包 foo 和子包 bar 以及模块 baz（当然也可以是个包）

这里有几点需要注意：

  1. 上面所说的限定名，也就是出现在 import xxx 句法或者是出现在 from xxx import yyy 句法里的 xxx 是一个指向脚本或者目录的路径名
  2. 限定名称的最后一部分首先会被视为脚本名，如果不存在的话才会被视为目录名，也即是说，会先尝试导入限定名称指定的模块
  3. 当使用 from ... import * 的方式导入时，如果定义了 __all__ 属性，则将会首先导入其中定义的名字（如上所述），否则将会导入模块中所有已经定义且不以 _ 开头的名字
  4. 当使用 from module import item 的方式导入时，首先会将 item 视为包中定义的名字尝试导入，没有的话才会尝试导入 item 模块

这里也有几个注意点：

  • sys.path 是一个可修改的列表，通常其给出的搜索路径包括：
    1. 被命令行直接运行的脚本文件所在目录，或者没有指定文件时的当前目录
    2. PYTHONPATH 环境变量指定的目录列表
    3. 标准库模块及其相关依赖模块所在目录
    4. 站点配置目录 site-packages
  • 在执行脚本之前先在缓存 sys.modules 中创建对应项的目的在于一定程度上避免循环导入
  • 缓存 sys.modules 可写，删除条目会导致下一次导入重新搜索，赋值为 None 则会强制下一次导入时抛错
  • 对于将找到的模块在当前作用域的名字空间中绑定到对应名称上：
    1. 如果使用 as 指定名字则使用所指定的名字
    2. 如果导入模块是最高层级模块则将使用该名字
    3. 如果导入模块不是最高层级模块，则将最高层级模块的名字绑定到当前名字空间，并且导入的模块必须使用完整的限定名称来访问

关于站点配置目录 site-packages
由 site 模块管理，这个模块在初始化 sys.path 时自动导入（可以命令行中使用 -S 选项来屏蔽）
站点配置目录通常位于 Python 安装目录的 lib 目录下，例如 /usr/lib/python3.6/site-packages（具体规则请查阅 官方文档）
除站点目录本身之外，还可以在目录中配置路径配置文件 name.pth，配置文件中的每一行会被视为 site-packages 的子目录被导入

importlib 中的一些工具

  1. importlib.__import__() 提供内置的 __import__() 函数的实现，通常不应该被手动调用
  2. importlib.import_module() 导入一个模块，是对 importlib.__import__() 的简单封装
  3. importlib.reload() 用于重新加载之前已经导入的模块
